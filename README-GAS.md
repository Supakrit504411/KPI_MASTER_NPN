# 🔌 วิธีเชื่อมต่อ Web App กับ Google Sheet (เพื่อบันทึกหมายเหตุ)

เว็บแอปนี้ดึงข้อมูลจาก Google Sheet อ่านได้ผ่าน public link อยู่แล้ว
แต่การ **เขียน/บันทึกหมายเหตุ** กลับไปที่คอลัมน์ K ต้องใช้ **Google Apps Script** เป็น backend
เพราะ Google Sheet ไม่อนุญาตให้เขียนผ่าน public link โดยตรง

---

## 📋 สิ่งที่ต้องมี

- ✅ Google Sheet ที่ตั้งค่า sharing เป็น "ทุกคนที่มีลิงก์สามารถดูได้"
- ✅ บัญชี Google ของคุณ (เจ้าของ Sheet)

---

## 🚀 ขั้นตอนที่ 1: วางโค้ด Apps Script

1. เปิด **Google Sheet** ของคุณ
2. ไปที่เมนู **Extensions → Apps Script**
   (จะเปิดหน้าต่างใหม่ของ Apps Script Editor)
3. ลบโค้ดเดิมในไฟล์ `Code.gs` ทิ้งทั้งหมด
4. **คัดลอกโค้ดทั้งหมด** จากไฟล์ `google-apps-script.gs` ในโฟลเดอร์นี้
5. วางลงใน Editor แล้วกด **Ctrl+S** เพื่อบันทึก (ตั้งชื่อ project อะไรก็ได้ เช่น "PEA Dashboard API")

---

## 🚀 ขั้นตอนที่ 2: Deploy เป็น Web App

1. ใน Apps Script Editor คลิก **Deploy → New deployment**
2. คลิกไอคอน **รูปเฟือง** ⚙️ แล้วเลือก **Web app**
3. ตั้งค่าดังนี้:

   | ช่อง | ค่าที่เลือก |
   |------|-----------|
   | **Description** | PEA Dashboard API |
   | **Web app** → Execute as | **Me** (your-email@gmail.com) |
   | **Web app** → Who has access | **Anyone** |

4. คลิก **Deploy**
5. จะมีหน้าต่างขึ้นให้ **Authorize Access** — คลิกแล้วเลือกบัญชี → คลิก **Advanced** → **Go to project (unsafe)** → **Allow**
   (เป็นเพราะ script ยังไม่ผ่านการ verify ของ Google แต่เป็นโค้ดของคุณเอง จึงปลอดภัย)
6. คัดลอก **Web app URL** ที่ได้ (จะมีรูปแบบ `https://script.google.com/macros/s/AKfyc.../exec`)

---

## 🚀 ขั้นตอนที่ 3: วาง URL ในเว็บแอป

1. เปิดไฟล์ `.env` ในโปรเจกต์ `pea-dashboard`
2. วาง URL ที่ได้ ลงในบรรทัด `VITE_APPS_SCRIPT_URL`

```env
VITE_GOOGLE_SHEET_ID=1P90pDKqos8bchlJRpr7GiQQJPEXdMIZvBps5BDVO1zE
VITE_GOOGLE_SHEET_NAME=LMS
VITE_APPS_SCRIPT_URL=https://script.google.com/macros/s/AKfycb.../exec
```

3. บันทึกไฟล์ แล้วรีสตาร์ท dev server (กด Ctrl+C แล้วรัน `npm run dev` ใหม่)

---

## ✅ ทดสอบ

1. เปิดเว็บแอปในเบราว์เซอร์ (http://localhost:5173)
2. ไปที่ตาราง "ตารางข้อมูลรายละเอียด"
3. คลิกไอคอน ▼ ด้านขวาของแถวที่ต้องการเพิ่มหมายเหตุ
4. พิมพ์ข้อความแล้วกดปุ่ม "💾 บันทึกหมายเหตุ"
5. ถ้าสำเร็จ จะมี toast สีเขียวขึ้นว่า "บันทึกหมายเหตุไปยัง Google Sheet สำเร็จ"
6. กลับไปดูที่ Google Sheet คอลัมน์ K จะมีข้อความที่พิมพ์

---

## 📝 ระบบเก็บ Log การกรอกหมายเหตุ (v2)

โค้ด `google-apps-script.gs` เวอร์ชันล่าสุด จะเก็บประวัติทุกครั้งที่มีการบันทึกหมายเหตุ
ลงชีตชื่อ **`NoteLogs`** ในไฟล์ Google Sheet เดียวกัน (สร้างให้เองอัตโนมัติครั้งแรก)

ข้อมูลที่เก็บต่อ 1 รายการ:

| คอลัมน์ | ความหมาย |
|---------|----------|
| วันเวลา (เวลาไทย) | เวลาที่บันทึกจริงฝั่ง server (Asia/Bangkok) ปลอมไม่ได้ |
| ผู้กรอก | ชื่อที่ผู้ใช้พิมพ์ในช่อง "ชื่อผู้บันทึก" (เว็บจำชื่อไว้ให้ ไม่ต้องพิมพ์ซ้ำ) |
| IP | Public IP ของเครื่องที่กรอก (จาก ipify) |
| การกระทำ | สร้างใหม่ / แก้ไข / ลบ / ไม่เปลี่ยนแปลง — ตัดสินจากค่าเดิมในเซลล์จริง |
| ชีต, แถว, หน่วยงาน (PEA), ข้อ | ระบุตำแหน่งว่ากรอกให้รายการไหน |
| ค่าเดิม → ค่าใหม่ | เห็นว่าแก้จากอะไรเป็นอะไร |
| เบราว์เซอร์/เครื่อง | userAgent ของเครื่องที่กรอก |

**สำคัญ:** ถ้าเคย deploy เวอร์ชันเก่าไว้แล้ว ต้องอัปเดตโค้ดใน Apps Script Editor
แล้วไปที่ **Deploy → Manage deployments → ✏️ Edit → Version: New version → Deploy**
(URL เดิมไม่เปลี่ยน ไม่ต้องแก้ .env)

ข้อจำกัดที่ควรรู้:
- ชื่อผู้กรอกเป็นการพิมพ์เอง (ไม่มีระบบล็อกอิน) จึงเชื่อได้ระดับหนึ่งเท่านั้น
- IP เป็น public IP — ถ้าหลายคนใช้เน็ตออฟฟิศเดียวกันจะเห็นเป็น IP เดียวกัน
- ถ้าดึง IP ไม่ได้ (เน็ตช้า/ถูกบล็อก) จะบันทึกว่า "(ไม่ทราบ IP)" แต่หมายเหตุยังบันทึกได้ปกติ

---

## 🔒 หมายเหตุด้านความปลอดภัย

- Apps Script จะทำงานในฐานะ **เจ้าของ Sheet** ("Execute as: Me")
- ตั้งค่า "Who has access: Anyone" เพราะเว็บแอปฝั่ง browser ไม่ได้ล็อกอิน
- หากต้องการจำกัดสิทธิ์ สามารถเพิ่ม token/password ใน `doPost()` ได้

---

## 🛠 ถ้ามีปัญหา

| ปัญหา | วิธีแก้ |
|------|--------|
| บันทึกแล้ว toast ขึ้น "เก็บไว้ในเครื่อง" | `VITE_APPS_SCRIPT_URL` ยังไม่ถูกตั้งค่า |
| บันทึกแล้ว error 401/403 | ตรวจสอบว่า deploy เป็น "Anyone" แล้ว |
| บันทึกแล้ว error 500 | ดูบรรทัดที่ error ใน Apps Script Editor → Executions |
| CORS error ใน console | ปกติ Apps Script รองรับ CORS แล้ว แต่ถ้าเจอ ลอง deploy ใหม่อีกครั้ง |
| ค่าไม่ขึ้นใน Sheet | ตรวจสอบว่าชื่อ sheet ตรงกับ `SHEET_NAME` ในไฟล์ .gs |
